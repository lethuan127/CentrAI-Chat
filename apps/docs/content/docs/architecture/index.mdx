## Tech Stack

| Layer | Technology | Rationale |
|-------|-----------|-----------|
| **Frontend** | Next.js 15 (App Router) | Full-stack React; SSR for SEO, client components for chat UI |
| **Backend** | NestJS 11 (TypeScript) | Modular, decorator-driven framework with built-in DI, guards, SSE streaming |
| **Database** | PostgreSQL + Prisma 7 | Type-safe schema, migrations, query building |
| **Realtime** | SSE over HTTP | Streaming LLM responses token-by-token; works through all proxies |
| **Queue** | Redis + BullMQ | Background jobs: title generation, audit writes |
| **Auth** | JWT + Passport.js | Local + OAuth (Google, GitHub); pluggable strategies |
| **Deployment** | Docker Compose | One-command deployment for dev and small teams |

## Main Modules

### Chat Service

Owns conversations and messages. Streams assistant responses via SSE.

- Create, list, archive, delete conversations
- Append user/assistant messages
- Stream responses via the AI SDK protocol
- Persist token usage metadata
- Emit events for audit logging

### Agent Service

Manages agent definitions and lifecycle. Admin/developer only.

- CRUD for agent definitions (prompt, model, tools, tags)
- Version management (immutable snapshots)
- Publish workflow: `draft` вҶ’ `published` вҶ’ `archived`
- Only published agents visible to end users

### Provider Adapter

Normalizes all LLM providers behind a single OpenAI-compatible interface.

- Unified `complete(messages, options)` across providers
- Streaming via SSE / chunked transfer
- Encrypted API key storage
- Health checks and connection testing
- Model sync from provider APIs

### Admin Module

Dashboard for system administration.

- User management (list, role assignment, deactivation)
- Analytics (conversations, messages, token usage)
- Audit log (who did what, when)
- System settings (default model, rate limits, feature flags)
